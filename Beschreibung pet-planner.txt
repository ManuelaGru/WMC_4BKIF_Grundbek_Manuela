Die Webseite besteht aus 3 HTML-Seiten, die alle dieselbe style.css verwenden und jeweils eigene JavaScript-Dateien laden:

index.html  →  weather.js + streak.js

Seite 1: Wetter & Aktivitäten

fetch – API-Anfrage
Das Herzstück der Seite. fetch() schickt eine Anfrage ans Internet und wartet auf eine Antwort:
javascript
// SCHRITT 1: Stadtname → Koordinaten (Geocoding)
const geoAntwort = await fetch(
  `https://geocoding-api.open-meteo.com/v1/search?name=Wien`
);
const geoDaten = await geoAntwort.json(); // Text → JavaScript-Objekt


// SCHRITT 2: Koordinaten → Wetterdaten

const wetterAntwort = await fetch(
  `https://api.open-meteo.com/v1/forecast?latitude=48.2&longitude=16.3`
);
const wetterDaten = await wetterAntwort.json();
async/await bedeutet: warten auf die Antwort bevor man weitermacht – weil das Internet Zeit braucht.

📅 Seite 2: Weekend Planner
Array mischen (Shuffle-Algorithmus)

javascript
function arrayMischen(array) {
  const kopie = [...array]; // Kopie erstellen

  // Fisher-Yates Algorithmus: von hinten nach vorne tauschen

  for (let i = kopie.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1)); // Zufälliger Index
    [kopie[i], kopie[j]] = [kopie[j], kopie[i]];   // Tauschen
  }
  return kopie;
}

// Ergebnis: jedes Mal eine andere Reihenfolge!

🏆 Gadget 1: Activity Streak (streak.js)

javascript
function streakBerechnen() {
  const aktivitaeten = JSON.parse(
    localStorage.getItem('petday_aktivitaeten') || '[]'
  );
  
  let streak = 0;
  let pruefDatum = new Date(); // Heute
  
  // Rückwärts durch die Tage gehen
  for (let i = 0; i < 365; i++) {
    const datumString = pruefDatum.toISOString().split('T')[0]; // "2026-05-21"
    
    if (aktivitaeten.includes(datumString)) {
      streak++; // Tag gefunden → zählen
    } else if (i > 0) {
      break;    // Lücke → aufhören
    }
    
    pruefDatum.setDate(pruefDatum.getDate() - 1); // Einen Tag zurück
  }
  return streak;
}

Idee & Ablauf – PetDay Planner

Die Idee

Viele Tierbesitzer fragen sich täglich: "Was machen wir heute?" – und das hängt stark vom Wetter ab. Die Idee war eine App zu bauen die Wetter, Tierinfos und Planung in einer Webseite kombiniert, damit Tierbesitzer schnell und einfach ihren Tag oder ihr Wochenende planen können.
Warum dieser Ablauf?

Seite 1 – Wetter zuerst
Das Wetter ist die Grundlage für alles. Erst wenn man weiß ob es warm, kalt oder regnerisch ist, kann man sinnvolle Aktivitäten vorschlagen. Deshalb ist die Wetterseite die Startseite.

Seite 2 – Planner danach
Wenn man das Wetter kennt, kann man vorausplanen. Der Wochenendplanner lässt den User sein Tier und seine Lieblingsmarken eingeben und generiert dann einen personalisierten Plan – das macht die App nützlich und individuell.

Seite 3 – Lexikon als Nachschlagewerk
Die dritte Seite ist informativer Inhalt – man kann Rassen entdecken, Favoriten speichern und tierfreundliche Orte in der Nähe finden. Das ergänzt die App mit echtem Mehrwert über den Alltag hinaus.

Die technische Logik dahinter
Der Ablauf folgt einem einfachen Prinzip:
Daten holen → Verarbeiten → Anzeigen
API (Internet) → fetch() → Array filtern → DOM aufbauen → User sieht Ergebnis
Jede Seite macht genau das, nur mit anderen Daten und anderem Ziel.
